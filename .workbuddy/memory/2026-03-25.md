## 出货利润计算修复（02:25）
- **问题**：旧逻辑用 `wageShare = wageTotal × (出货量/总入库量)` 按比例分摊薪资，导致利润错误
  - 例：批次入库100件，薪资2元，第一次出货50件，算出利润=49而非48
- **修复方案**：
  1. 出货弹窗新增「本次员工薪资」输入框，用户直接填写本次对应薪资，利润 = 销售额 - 手动输入薪资
  2. `doOutBatch(batchId, qty, forceClose, manualWage)` 新增第四参数
  3. 新增「重算历史利润」按钮，弹出所有出货记录列表，允许逐条修改薪资并重新保存
- **出货弹窗薪资提示**：自动计算并展示「批次总薪资/已分摊/待分摊」作为参考值

## 前端 localStorage → API 全面改造完成（15:52）

### 改造内容（index.html）
1. **delRecord / editRecord**：接收 `recId`（数据库id），调 `DELETE/PUT /api/records/:id`；`adminLoadRecord` 渲染改为传 `rec.id`
2. **adminAddProduct**：async，调 `POST /api/companies/:company/products`
3. **adminDelProduct**：async，调 `DELETE /api/companies/:company/products/:key`
4. **addProcess**：async，调 `POST /api/products/:key/processes`，返回 `id` 存入本地缓存
5. **saveProc**：async，用本地缓存的 `proc.id` 调 `PUT /api/products/:key/processes/:id`
6. **delProc**：async，用本地缓存的 `proc.id` 调 `DELETE /api/products/:key/processes/:id`
7. **adminLoadSummary**：改 async，完全替换为调 `/api/companies/:company/summary`
8. **backupData**：调 `GET /api/companies/:company/export` 下载备份 JSON
9. **restoreData**：读文件后 `POST /api/companies/:company/import` 恢复（新增后端接口）
