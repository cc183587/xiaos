# 数据库备份说明

## 备份位置
```
factory-backend\backups\
```
备份文件命名格式：`factory_backup_YYYY-MM-DD_HH-MM-SS.db`

## 备份方式

### 1. 手动备份（立即备份）
双击运行 **`备份数据库.bat`**

### 2. 自动定时备份
设置系统每天自动备份（默认凌晨 02:00）

**步骤**：
1. 右键点击 **`setup-backup-task.bat`**
2. 选择 **"以管理员身份运行"**
3. 等待提示"任务计划创建成功"

**修改备份时间**：
```cmd
schtasks /change /tn "工厂系统数据库备份" /st 18:00
```
（改为每天 18:00 备份）

**删除自动备份任务**：
```cmd
schtasks /delete /tn "工厂系统数据库备份" /f
```

### 3. 查看任务计划
打开 Windows 任务计划程序查看：
```
控制面板 → 管理工具 → 任务计划程序
```
找到任务：`工厂系统数据库备份`

## 备份策略
- **保留数量**：最近 7 个备份
- **备份周期**：每天（可自定义）
- **自动清理**：超过 7 个的旧备份会自动删除

## 恢复数据库
如果需要恢复数据库，只需将备份文件复制回原位置：

```cmd
copy factory-backend\backups\factory_backup_2026-03-25_14-30-00.db factory-backend\database\factory.db
```

**注意**：恢复前先停止后端服务（关闭后端窗口）

## 备份检查
定期检查备份是否正常生成：
1. 打开 `factory-backend\backups\` 目录
2. 确认有最新的备份文件
3. 检查文件大小是否正常（应与源文件接近）

## 建议备份策略
- **日常**：每天自动备份（凌晨 02:00）
- **重要操作前**：手动备份一次
- **月底/月底**：额外备份并存储到外部存储（U盘/云盘）

## 注意事项
1. 备份需要数据库文件存在才能执行
2. 备份过程中不要关闭后端服务
3. 定期检查备份完整性
4. 重要数据建议多份备份
